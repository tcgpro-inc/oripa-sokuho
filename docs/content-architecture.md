# コンテンツ管理設計書

## 基本方針: MDファイルベース管理

記事コンテンツはMarkdownファイルとしてGitリポジトリ内で管理する。
DBは初期段階では使用せず、必要になった時点（コメント機能・リアルタイムランキング等）で導入する。

### 設計思想

1. **ファイル = 記事** — 1つのMDファイルが1つの記事に対応する。PHPのようなシンプルさ。
2. **Git = CMS** — 編集履歴はGitが担う。レビューはPRで行える。
3. **Claude Code = 編集者** — データ取得 → 記事生成 → pushまでをClaude Codeが担当。
4. **ビルド時インデックス** — frontmatterからカテゴリ・タグ・日付のインデックスをビルド時に生成。
5. **段階的拡張** — 動的機能（コメント、PVランキング等）が必要になった段階でDBを追加する。

## ディレクトリ構成

```
content/
  articles/
    2026-03-31-dapa-oripa-new-gacha.md
    2026-03-31-clove-oripa-campaign.md
    ...
```

### ファイル命名規則

```
{YYYY-MM-DD}-{slug}.md
```

- 日付はISO形式（YYYY-MM-DD）
- slugは英数字とハイフンのみ（日本語不可）
- 短く、内容が分かる名前にする

## Frontmatter仕様

```yaml
---
title: "記事タイトル"           # 必須。【速報】等のタグを含む
tag: "速報"                     # 必須。速報 / 朗報 / 悲報 / 炎上 / まとめ
category: "pokeka"              # 必須。pokeka / yugioh / onepiece / other
source_url: "https://..."       # 必須。情報ソースのURL
source_name: "PRTIMES"          # 必須。PRTIMES / X / 公式サイト / YouTube / その他
company: "企業名"               # 任意。プレスリリース元の企業名
thumbnail: "/images/articles/xxx.webp"  # 任意。サムネイル画像
published_at: "2026-03-31T10:00:00+09:00"  # 必須。公開日時（ISO 8601）
hot: false                      # 任意。HOT!バッジ表示（デフォルト: false）
new: true                       # 任意。NEW!バッジ表示（デフォルト: true、3日経過で自動false）
---
```

```markdown
---
(frontmatter)
---

記事の導入文。1〜2文でニュースの概要を伝える。

本文の詳細情報。

[元記事を読む](https://source-url)
```

## 運用フロー

### 通常運用（Claude Code利用）

```
① Claude Codeに記事作成を依頼
② Claude CodeがPRTIMES等からデータ取得
③ MDファイルを content/articles/ に生成
④ 内容を確認・必要に応じて編集
⑤ git push → Vercel自動デプロイ → 公開
```

### 将来の自動化（Phase 2以降）

```
① GitHub Actions（Cron: 毎日9時/18時）
② スクリプトがPRTIMES APIを叩いて新着取得
③ MDファイルを自動生成 → PRを自動作成
④ 人間がPRをレビュー → マージで公開
```

## カテゴリ定義

| slug | 表示名 | 対象 |
|------|--------|------|
| `pokeka` | ポケカ | ポケモンカード関連のオリパ |
| `yugioh` | 遊戯王 | 遊戯王カード関連のオリパ |
| `onepiece` | ワンピース | ワンピースカード関連のオリパ |
| `other` | その他 | 上記以外のTCGオリパ、業界全体の話題 |

## タグ定義

| タグ | 用途 | 記事タイトル例 |
|------|------|---------------|
| `速報` | 新サービス・新機能・キャンペーン開始 | 【速報】DAPAオリパ、ポケカ新ガチャ追加 |
| `朗報` | ユーザーにとって良いニュース | 【朗報】〇〇オリパ還元率80%超えガチャ登場 |
| `悲報` | サービス終了・改悪など | 【悲報】〇〇オリパ、サービス終了を発表 |
| `炎上` | 不正疑惑・トラブル | 【炎上】〇〇オリパの確率表記に疑惑 |
| `まとめ` | 定期まとめ・比較記事 | 【まとめ】2026年3月のオリパ新サービス一覧 |

## DBへの移行タイミング

以下の機能が必要になった段階でNeon (PostgreSQL) を導入する:

- **コメント機能** — ユーザーの書き込みを保存する必要がある
- **PVベースのランキング** — リアルタイムのアクセス数集計が必要
- **ユーザー投稿** — 外部からのコンテンツ投稿を受け付ける
- **通知機能** — ユーザーごとの設定を保存する必要がある

移行時は、既存のMDファイルをDBにインポートするマイグレーションスクリプトを作成する。
MDファイルのfrontmatter仕様がそのままDBスキーマの元になるため、移行コストは低い。
